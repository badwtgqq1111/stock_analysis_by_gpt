# 批量保存数据功能指南

## 功能说明

本项目新增了批量保存数据功能，支持处理多个指定的股票并验证数据是否成功保存到数据库。

### 核心功能

1. **单股票处理并验证** - 处理单只股票并验证数据
2. **多股票批量处理** - 一次指定多个股票代码进行批量处理
3. **数据验证报告** - 处理完成后生成详细的数据验证报告
4. **增量更新支持** - 自动检测数据库中的现有数据并进行增量更新

## 使用方法

### 1. 单股票处理（带验证）

**处理单只股票并验证数据是否存在数据库中：**



## 命令行参数详解

| 参数 | 说明 | 示例 |
|------|------|------|
| `stock_code` | 单只股票代码（可选，默认03633） | `03633` |
| `--stocks` | 逗号分隔的多股票代码列表 | `--stocks "03633,02590"` |
| `--all` | 处理全市场所有港股 | `--all` |
| `--limit` | 限制处理股票数量（与--all一起使用） | `--all --limit 50` |
| `--verify` | 完成处理后验证数据 | `--verify` |
| `--no-chart` | 禁用K线图生成 | `--no-chart` |
| `--output` | 输出目录路径 | `--output ./my_output` |
| `--db` | 数据库目录 | `--db ./my_db` |

## 实现细节

### 增量更新机制

- 首次处理某个股票时，会从数据库检查是否已有数据
- 如果数据库中已有该股票的数据，会自动检测最新日期
- 新数据会与现有数据合并（重复日期会被更新）
- 每次更新操作都会在 `update_log` 表中记录

### 数据保存流程

1. **获取数据** → 从腾讯财经 API 获取股票数据
2. **保存到数据库** → 使用 DuckDB 进行高性能批量保存
3. **导出JSON** → 同时生成 JSON 文件备份
4. **验证数据** → 如果指定了 `--verify`，会验证数据是否成功保存

### 验证算法

验证过程检查：
- 股票代码是否存在数据库中
- K线数据记录数
- 时间范围覆盖情况
- 股票基本信息是否已保存
- 数据库文件大小

## 常见问题

### Q: 如何知道数据是否成功保存到数据库？

A: 使用 `--verify` 参数，完成处理后会生成详细的验证报告。

### Q: 什么时候会进行增量更新？

A: 当数据库中已有该股票的数据时，会自动进行增量更新。新数据会追加，相同日期的数据会被更新。

### Q: 可以同时处理多少只股票？

A: 理论上无限制，但受网络和时间限制。建议每次不超过100只。

### Q: 如何跳过图表生成来加快处理？

A: 使用 `--no-chart` 参数可以跳过K线图生成，加快处理速度。

## 技术细节

- **数据库** → DuckDB（列式存储，OLAP优化）
- **批量操作** → 使用临时表和UPSERT操作进行高效批量更新
- **增量更新** → 自动检测最新日期和UNIQUE约束实现
- **数据持久化** → JSON相关、DuckDB表结构完全持久化
